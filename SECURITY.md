# Security Policy

## Supported Version

Only the latest tagged release is supported during the initial development phase.

## Reporting

Do not include company data, credentials, internal hostnames, or exploit payloads containing real secrets in a public issue. Use GitHub private vulnerability reporting when enabled.

## Runtime Boundary

This repository is not a security certification. Company deployment requires IT review of artifacts, endpoints, firewall rules, dependency inventory, logging, backup, and rollback.
