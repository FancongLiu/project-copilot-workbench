# SOP-SYN-003: synthetic configuration change control

Every proposed parameter change needs a decision ID, approval date, effective date, owner, and read-only implementation evidence. The decision register records whether the proposal is implemented, deferred, or superseded.

For conflicting values, answer with all relevant dated values and identify the current state only when implementation evidence exists. If evidence is missing, ask for clarification or state that the project sources are insufficient.

Never treat a meeting proposal as implemented merely because it is newer than a baseline. Never send commands to physical equipment from this software.
